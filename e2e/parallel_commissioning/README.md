# e2e_parallel_commissioning

Эмпирически измеряет, насколько параллельный commissioning по нескольким DALI-шинам
одного шлюза быстрее последовательного, и насколько эта разница стабильна от
итерации к итерации.

## Стенд

- **Один** WB-DALI-шлюз.
- На каждой шине подключены реальные DALI-устройства.
  Состав устройств на шинах может различаться.
- `wb-mqtt-serial` запущен и обслуживает шлюз.

`wb-mqtt-dali` останавливается в начале прогона и перезапускается после завершения.

## Что измеряется

### Reference run

Последовательно для каждой шины: сброс коротких адресов и настроек устройств
через широковещательные запросы потом вызов сканирования и назначения адресов.
Замеряется время сканирования и назначения адресов и фиксируется число найденных 
устройств — это **референсное количество устройств** для этой шины на оставшуюся часть теста.

### Parallel iterations

`--iterations` раз (по умолчанию 10) повторяется:

1. Reset на всех шинах.
2. Сканирование запускается одновременно на всех шинах.
   Для каждой шины замеряется собственное время прогона —
   оно идёт в гистограмму **этой шины**.
3. Найденное количество устройств сверяется с референсом этой шины.
   Несовпадение → итерация на этой шине помечена `mismatch`.
   Исключение при сканировании → `exception`.
   Шины независимы: упавшая шина не отменяет итерацию для остальных.

## Запуск

```
.venv/bin/python e2e/parallel_commissioning/e2e_parallel_commissioning.py
```

Опции:

| Флаг | По умолчанию | Назначение |
| --- | --- | --- |
| `--gateway` | `wb-dali_1` | MQTT device-id шлюза с несколькими шинами |
| `--iterations` | `10` | Сколько раз повторить параллельный скан |
| `-d`, `--debug` | off | DEBUG-логи |

## Вывод

В конце прогона печатается:

1. **Таблица reference-времён** — по строке на шину: номер, время в секундах,
   количество устройств.
2. **По каждой шине** — гистограмма времён **успешных** параллельных итераций
   и счётчики:
   - `successful` — итераций без ошибок (попали в гистограмму);
   - `mismatched` — итераций, где число найденных устройств не совпало
     с референсом;
   - `exceptions` — итераций с исключением в reset или при сканировании.
