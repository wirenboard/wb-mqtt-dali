# wb-mqtt-dali

DALI-MQTT bridge which follows [Wiren Board MQTT Conventions](https://github.com/wirenboard/conventions/blob/main/README.md).
It's designed to be used on [Wiren Board](https://wirenboard.com/en/) family of programmable automation controllers.

Works only with Wiren Board's [WB-DALI](https://wiki.wirenboard.com/wiki/WB-DALI_Modbus-to-DALI_Gateway_\(3-channel\)) Modbus-to-DALI gateway modules. `wb-mqtt-dali` talks to them over MQTT-RPC via [wb-mqtt-serial](https://github.com/wirenboard/wb-mqtt-serial) and does not access the serial port directly.

Драйвер шины DALI / DALI-2 для Wiren Board. Публикует обнаруженные на шине устройства как виртуальные устройства WB в MQTT согласно [Конвенции Wiren Board MQTT](https://github.com/wirenboard/conventions/blob/main/README.md) и принимает команды для управления светильниками, цветовой температурой, цветом и т. п.

Работает только с шлюзами Wiren Board [WB-DALI](https://wiki.wirenboard.com/wiki/WB-DALI_Modbus-to-DALI_Gateway_\(3-channel\)). `wb-mqtt-dali` обращается к ним по MQTT-RPC через [wb-mqtt-serial](https://github.com/wirenboard/wb-mqtt-serial) и не имеет прямого доступа к последовательному порту.

## Структура конфигурационного файла

Файл `/etc/wb-mqtt-dali.conf`. Минимальный валидный пример:

```jsonc
{
  // Включить отладочный вывод
  "debug": false,

  // Список WB-DALI шлюзов. Каждый шлюз имеет три шины.
  "gateways": [
    {
      // MQTT device_id WB-DALI — тот же id, под которым модуль
      // опубликован сервисом wb-mqtt-serial
      "device_id": "<wb-dali device id from wb-mqtt-serial>",

      // Включение WebSocket-сервера для Lunatone Cockpit
      "websocket_enabled": true,
      "websocket_port": 8080,

      // Физические DALI-шины
      "buses": [
        {
          // Период опроса состояния устройств, в секундах.
          "polling_interval": 5,

          // Включение публикации всех DALI-пакетов на шине в отдельный MQTT-топик
          // /wb-dali/<uid>/bus_monitor
          "bus_monitor_enabled": false,

          // Дублировать строки монитора в лог контроллера. Действует только при bus_monitor_enabled.
          "bus_monitor_syslog_enabled": false,

          // Известные устройства на шине. Заполняется автоматически
          // во время поиска на шине
          "devices": [
            {
              "short": 0,             // короткий адрес 0..63
              "random": 12345678,     // случайный адрес
              "dali2": true,          // true, если устройство поддерживает DALI-2
              "mqtt_id": "my-lamp-1", // необязательно: переопределить MQTT device_id (иначе генерируется автоматически)
              "name": "Лампа в холле" // необязательно: отображаемое имя устройства
            }
          ]
        }
      ]
    }
  ]
}
```

## Топики MQTT

Каждое устройство публикуется как виртуальное устройство Wiren Board в `/devices/<mqtt_id>/...` по [конвенции WB MQTT](https://github.com/wirenboard/conventions/blob/main/README.md). Здесь `<mqtt_id>` — идентификатор устройства (поле `mqtt_id` в конфиге), его не следует путать с `device_id` шлюза. Человекочитаемое имя устройства, отображаемое в интерфейсе, задаётся полем `name` и публикуется в `/devices/<mqtt_id>/meta/name`.

Состав control'ов зависит от типа устройства и его возможностей: яркость, минимальный и максимальный уровни, цветовая температура или цвет, статусы ошибок и т. п. Например, LED-модуль (DT6) с управлением цветом (DT8) публикует control'ы яркости, диапазона уровней и цвета/цветовой температуры.

Кроме самих устройств, для каждой шины публикуются два служебных виртуальных устройства:

- **broadcast** — отправка команд сразу всем устройствам шины (broadcast-адрес); состояние не отражается.
- **group** — по одному на каждую DALI-группу (0..15) для группового управления.

Если `mqtt_id` и `name` у устройства в конфиге не заданы, они генерируются автоматически из идентификатора шины `<device_id>_bus_<N>` (где `<device_id>` — поле шлюза в конфиге, `<N>` — номер шины 1..3) и короткого адреса:

| Устройство          | `<mqtt_id>` по умолчанию            | Имя по умолчанию            |
| ------------------- | ----------------------------------- | --------------------------- |
| DALI (control gear) | `<device_id>_bus_<N>_<short>`       | `DALI <short>`              |
| DALI-2              | `<device_id>_bus_<N>_dali2_<short>` | `DALI-2 <short>`            |
| broadcast           | `<device_id>_bus_<N>_broadcast`     | `Bus <N> широковещательный` |
| group               | `<device_id>_bus_<N>_group_<MM>`    | `Bus <N> группа <M>`        |

где `<short>` — короткий адрес (0..63), `<M>` — номер группы (0..15), `<MM>` — он же с ведущим нулём (`00`..`15`). `mqtt_id` и `name` в конфиге переопределяют эти значения (broadcast и group в конфиге не задаются).

### Монитор шины

При включении параметра `bus_monitor_enabled` весь DALI-трафик шины публикуется построчно в диагностический топик `/wb-dali/<uid>/bus_monitor` (`<uid>` — идентификатор шины).

Формат строки: `{HH:MM:SS.mmm} {напр}{hex} {расшифровка}[ - {ответ}]`, где:

- `{HH:MM:SS.mmm}` — время;
- `{напр}` — направление: 
  - `>>` — запрос отправлен `wb-mqtt-dali`; 
  - `<<` — неожиданный пакет принят с шины;
- `{hex}` — сырое значение пакета;
- `{расшифровка}`:
  - для распознанной команды это выражение `Name(A<n>, …)` в синтаксисе RPC [`Bus/SendCommand`](#bussendcommand) (строку можно скопировать и отправить обратно);
  - для нераспознанного пакета — дескриптор `FF{len}` (forward) или `BF{len}` (backward);
- `{ответ}` — добавляется только к запросу `wb-mqtt-dali`, у которого есть ответ: либо `{hex} {значение}` backward‑пакета, либо текст статуса ошибки (см. таблицу статусов ниже).

Суффиксы: 
  - `(fc: N)` — счётчик пакетов аппаратного монитора (только неожиданных пакетов);
  - `(from lunatone)` — команда пришла из эмулятора Lunatone Cockpit.

Варианты строк:

| Что | Пример |
| --- | --- |
| Запрос `wb-mqtt-dali` с ответом-значением | `12:34:56.812 >> a3fa QueryActualLevel(A5) - 00fe 254` |
| Запрос `wb-mqtt-dali`, не подразумевающий ответа | `12:34:56.900 >> a3a0 Off(A5)` |
| Запрос `wb-mqtt-dali` с ошибкой в ответе | `12:34:56.950 >> a3fa QueryActualLevel(A5) - no power on bus` |
| Команда из эмулятора Lunatone | `12:34:57.000 >> a3a0 Off(A5) (from lunatone)` |
| Неожиданная команда или событие с шины | `12:34:57.001 << a380 DAPC(A3, 128) (fc: 42)` |
| Неожиданный нераспознанный forward-пакет с шины | `12:34:57.050 << ff93 FF16 (fc: 43)` |
| Неожиданный нераспознанный backward-пакет с шины | `12:34:57.100 << 002a BF8 (fc: 44)` |
| Неожиданный пакет с шины, принятый с ошибкой | `12:34:57.150 << ff93 FF16 framing error (fc: 45)` |

Тексты статусов в позиции `{ответ}` — приходят от шлюза вместо данных, когда передача не удалась:

| Статус | Значение |
| --- | --- |
| `no response from gateway` | шлюз не прислал ответ на команду (таймаут) |
| `no transmission, internal gateway error` | шлюз не выполнил передачу из-за внутренней ошибки |
| `no power on bus` | на шине нет питания |
| `gateway overheated` | перегрев шлюза — передача временно заблокирована |
| `transmission cancelled` | передача отменена по инициативе wb-mqtt-dali |
| `gateway unavailable` | шлюз недоступен |
| `unknown response status` | шлюз вернул нераспознанный код статуса |
| `transmission error` | не удалось разобрать ответ от шлюза |
| `framing error` | ошибка приёма пакета ответа |

При `bus_monitor_syslog_enabled` те же строки дополнительно пишутся в лог контроллера. В лог строка идёт **без** метки времени `{HH:MM:SS.mmm}`, MQTT-публикация при этом не меняется.

## MQTT-RPC API

### Общий формат запроса и ответа

Запрос отправляется в топик `/rpc/v1/wb-mqtt-dali/<Namespace>/<Method>/<clientId>`, где `clientId` — произвольный идентификатор клиента (обычно UUID). Ответ приходит в одноимённый топик с суффиксом `/reply`. Полезная нагрузка — JSON.

Запрос:

```jsonc
{
  // Порядковый номер запроса, копируется в ответ как есть.
  "id": 1,

  // Параметры конкретного метода. Может быть пустым объектом.
  "params": {}
}
```

Ответ:

```jsonc
{
  // Совпадает с id запроса.
  "id": 1,

  // Результат метода — объект или массив. Присутствует только при успехе.
  "result": {},

  // null при успехе; при ошибке — объект { code, message, data }.
  "error": null
}
```


### Bus/ListCommands

Топик: `/rpc/v1/wb-mqtt-dali/Bus/ListCommands/<clientId>`

Вернуть полный каталог DALI-команд, известных серверу, в виде, удобном для автодополнения в редакторе (CodeMirror).

#### Параметры
Метод не принимает параметров.

#### Ответ


```jsonc
[
  {
    // Имя команды, как пишется в SendCommand (например, "DAPC", "DT8.Activate").
    "name": "DAPC",

    // Категория для группировки (например, "Gear", "DT6", "FF24").
    "category": "Gear",

    // Шаблон вставки с метками ${N:placeholder} для табстопов CodeMirror.
    "snippet": "DAPC(${1:A0}, ${2:0xFE})"
  },
  ...
]
```

### Bus/SendCommand

Топик: `/rpc/v1/wb-mqtt-dali/Bus/SendCommand/<clientId>`

Выполнить произвольную последовательность DALI-команд на указанной шине **атомарно**.

Каждая команда — строка вида `ИМЯ_КОМАНДЫ(аргумент, ...)`.
Аргументы:
- `A<n>` — короткий адрес (0..63),
- `G<n>` — группа (для control gear 0..15, для control device 0..31),
- `I<n>` — номер компонента DALI-2 устройства (0..31),
- данные в виде числа в шестнадцатеричной (`0xHEX`) или десятичной записи.

Если у команды нет аргументов, можно не добавлять пустые скобки. Команды, принимающие адрес, без его указания отправляются широковещательным запросом.

#### Параметры

```jsonc
{
  // UID шины. Обязательный.
  "busId": "wb-mdali_1_bus_1",

  // Непустой массив строк-команд. Обязательный.
  "commands": ["DAPC(A5, 0xFE)", "QueryActualLevel(A5)"]
}
```

#### Ответ

Массив ответов на команды в том же порядке, что и в запросе.

```jsonc
[
  {
    // "ok" или "error"
    "status": "ok",
  
    // Только при status == "ok" для команды которая подразумевает ответ
    "response": {
      "raw": 254, 
      "value": "254" 
    },
  
    // Только при status == "error": описание ошибки
    "error": "timeout"
  },
  ...
]
```

#### Пример

Запрос:
```json
{
  "id": 7,
  "params": {
    "busId": "wb-mdali_1_bus_1",
    "commands": ["DAPC(A5, 0xFE)", "QueryActualLevel(A5)"]
  }
}
```

Ответ:
```json
{
  "id": 7,
  "result": [
    {"status": "ok"},
    {"status": "ok", "response": {"raw": 254, "value": "254"}}
  ],
  "error": null
}
```
